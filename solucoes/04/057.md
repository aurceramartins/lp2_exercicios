## Problema

57 - Que princípios gerais de programação por objetos é que estão a ser
violados no seguinte diagrama UML? Justifica a tua resposta.

![UML](04/057.png)

## Soluçoes

### Solução 1


- Viola princípio Open/closed Classes devem estar abertas para extensão mas fechadas à modificação.
	Se quiseras ampliar o codigo e introducir outros tipos de Jogadores como por exemplo um troll
	terias que modificar a clase Player.

- Viola o Dependency inversion principle: Devemos depender de abstrações, não de concretizações.
	Devemos programar para o tipo mais abstrato possível: Classe base, classe base abstrata, interface
	não para uma implementação.
